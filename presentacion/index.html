01 · Comprensión del requerimiento20 pts

De solo contado
a vender en cuotas.

GQG System solo facturaba al contado. Con el negocio creciendo, los clientes pedían pagar en cuotas y había que rechazar la venta o llevarla a mano, con riesgo de error. La solución: elegir la modalidad de pago al registrar el comprobante y que el sistema calcule los vencimientos solo.

Lo que faltaba no era la cabecera de la factura —ya existía— sino el detalle de cuotas: número, importe, vencimiento, cobrado.

CO · CONTADO

Contado

Una sola cuenta, vence el mismo día de la factura.

CR · REGULAR

Crédito Regular

Cuotas mensuales fijas: i × 30 días desde la factura (30/60/90).

CR · IRREGULAR

Crédito Irregular

El usuario define los días de cada cuota (30/45/60).

04 · Implementación técnica — el núcleo15 pts

ins_ventas
AFTER INSERT ON VENTAS

Valida tipo y plazo

Si el tipoid del documento no coincide con el del plazo → SIGNAL SQLSTATE '45000'. No se registra un crédito con plazo de contado.

Exige cuotas > 0

Si nCuotas = 0 → SIGNAL. Lee también si el plazo es irregular.

Trunca por moneda

importe = TRUNCATE(total / cuotas, decimales) — Guaraní = 0.

Última cuota absorbe el redondeo

última = total − base × (cuotas−1). La suma cuadra exacto.

Vencimientos con DATE_ADD

regular i×30; irregular toma PLAZO_DETALLES.dias. Inserta una fila por cuota.

-- 1) tipo de documento y plazo deben coincidir
IF doctipoid <> plazotipoid THEN
  SIGNAL SQLSTATE '45000'
    SET MESSAGE_TEXT = 'Tipo de documentos y plazos no coinciden.';
END IF;

-- 2) importe base + última cuota (cuadre exacto)
SET importecuota = TRUNCATE(NEW.totalfactura / nCuotas, decimalesMon);
SET ultimacuota  = NEW.totalfactura - importecuota * (nCuotas - 1);

-- 3) una fila de cuota por iteración
IF isIrregular = 1 THEN
   SELECT IFNULL(pd.dias, i*30) INTO dias FROM PLAZO_DETALLES pd
    WHERE pd.plazoid = NEW.plazoid AND pd.cuota = i;
ELSE SET dias = i * 30; END IF;

INSERT INTO CUENTAS_COBRAR(tabla,tablaid,cuota,importe,cobrado,vence)
 VALUES('VENTAS',NEW.id,i,imp,0,
        DATE_ADD(NEW.fechafactura,INTERVAL dias DAY));

Frontend · diseñoDESIGN.md

Las cuotas, antes de confirmar

Selector de cliente, fecha, productos y un toggle Contado / Crédito que filtra los plazos válidos. Al elegir el plazo, las cuotas se previsualizan al instante —numeradas 01/03, fechas DD/MM/AAAA— con un indicador de que la suma cuadra con el total. Es el reflejo visual de lo que hace el trigger.

Control de secuencia: CONFIRMAR VENTA queda deshabilitado hasta generar las cuotas. El total es derivado de las líneas, no se tipea a mano.
